 # Business Discovery Engine Seed
 
 This is the condensed seed input for the new planning system.
 
 ## Core Goal
 
 Design a system to research, compare, and shortlist service businesses that fit a specific founder thesis, then later build an AI-assisted pipeline to run that research repeatedly.
 
 ## Founder Thesis
 
 The target business should ideally have:
 
 - a very large customer pool
 - high-margin service economics
 - a lean team made of skilled/professional workers or no workers
 - tolerance for boring/simple markets if economics are strong
 - little to no dependence on selling commodity products
 
 ## Strategic Intent
 
 The plan should first help decide what business categories are worth pursuing.
 
 Only after the requirements, metrics, decision process, and research workflow are mature should the planning move into technical implementation of an AI-enabled discovery pipeline.
 
 ## Desired Planning Sequence
 
 1. Product-manager style planning
 - problem framing
 - goals
 - constraints
 - evaluation criteria
 - stages
 - approval checkpoints
 
 2. Research-backed decision design
 - what data to gather
 - what signals matter
 - how options are compared
 - what must be reviewed manually
 
 3. Technical planning
 - how AI agents participate
 - what files/context they use
 - how they hand off work
 - how they return findings to the origin chat
 - how the final implementation plan is created with human supervision
 
 4. Execution handoff
 - once the plan is complete enough, create a clean handoff package for build agents
 
 ## Process Requirements
 
 - the plan must be built conversationally with strong human intervention
 - important instructions must be converted into semantic memory/context files
 - multiple chats/agents must stay connected through explicit files and return loops
 - the system should be reusable for other future planning efforts

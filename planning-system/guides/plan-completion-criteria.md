 # Plan Completion Criteria
 
 A plan is complete enough for execution handoff only when all are true:
 
 - the problem framing is stable
 - goals and non-goals are explicit
 - requirements and decision metrics are defined
 - stage structure and approval points are clear
 - open knots have no blockers left
 - technical implementation planning is mature enough to guide build agents
 - a handoff package can be created without relying on chat history alone
- clean plan-only artifacts are up to date and separated from memory/context artifacts
 
 ## Required Final Outputs
 
 - product/strategy plan
 - technical implementation plan
 - memory/context package
 - execution handoff brief

Suggested plan artifact paths per project run:

- `project-runs/<project-slug>/plans/product-strategy-plan.md`
- `project-runs/<project-slug>/plans/technical-implementation-plan.md`
 
 ## Handoff Decision Format
 
 The origin chat should state:
 
 - completion status: ready / not ready
 - blockers remaining
 - what is approved
 - what is still optional
 - recommended first execution agent
